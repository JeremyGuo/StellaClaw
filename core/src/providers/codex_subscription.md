# codex_subscription ChatMessage Translation

This provider translates `List<ChatMessage>` into OpenAI Responses-style `input` items for the Codex subscription websocket.

## Capability Gates

- `image_in`: user image files can become `input_image`. Tool-result images and assistant-generated images are replayed as temporary user image messages.
- no `image_in`: image files are converted to text references by `normalize_messages_for_model`, or assistant image outputs are sent as `output_text` references.
- `pdf_in` / `audio_in` / `file_in`: handled by `normalize_messages_for_model` before provider-specific serialization. If unsupported, files become text references.
- `image_out`: affects generic model normalization for assistant image history, but Codex-specific assistant image replay now relies on `image_in`.

## User ChatMessage

`role=user` becomes:

```json
{
  "type": "message",
  "role": "user",
  "content": [...]
}
```

Item mapping:

- `Context`: `{ "type": "input_text", "text": context.text }`
- `File` image: `{ "type": "input_image", "image_url": file.uri }`
- `File` non-image:
  - `data:` URI: `{ "type": "input_file", "filename": ..., "file_data": ... }`
  - OpenAI file id (`file-*` / `file_*`): `{ "type": "input_file", "file_id": ... }`
  - local `file://`: read and inline as `file_data`
  - otherwise: `{ "type": "input_file", "file_url": file.uri }`
- `ToolCall`: text fallback as `<tool_call name="...">...</tool_call>`
- `Reasoning` and `ToolResult`: ignored in the user message body

After a user message body, any `ToolResult` items in the same `ChatMessage` are appended as top-level `function_call_output` items. If a `ToolResult` carries an image and the model supports image input, a temporary user message is appended after the function output with that image as `input_image`.

## Assistant ChatMessage

Codex-only encrypted `Reasoning` is replayed first as top-level Responses `reasoning` items. Plain reasoning is dropped by provider normalization.

Assistant content items are accumulated into assistant `message` items:

- `Context`: `{ "type": "output_text", "text": context.text }`
- non-image `File`: `{ "type": "output_text", "text": file.uri }`
- image `File` with `state == None`:
  - with `image_in`: flush current assistant text, then append a temporary user message:
    ```json
    {
      "type": "message",
      "role": "user",
      "content": [
        { "type": "input_text", "text": "[Generated by Assistant]" },
        { "type": "input_image", "image_url": "..." }
      ]
    }
    ```
  - without `image_in`: append an `output_text` image reference
- `ToolCall`: flush current assistant text, then append top-level `function_call`
- `ToolResult`: ignored in assistant content, then appended as `function_call_output` after content

## Response Back To ChatMessage

Responses `message` assistant content becomes `Context` or `File`; `reasoning` becomes Codex `Reasoning`; `function_call` becomes `ToolCall`; `image_generation_call.result` is materialized into a `FileItem`.
