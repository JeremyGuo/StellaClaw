结论

当前 Web Channel 设计整体可用：REST 负责快照/管理/二进制传输，WebSocket 负责 conversation list、foreground session 和 terminal 的实时数据面。状态事实源已进一步收敛：列表和 conversation 级状态只消费 conversation stream，当前会话 foreground stream 只承载消息、turn progress 和错误。
推荐事实源左侧列表和 conversation 级状态使用 /api/conversations/stream；当前会话消息和 turn progress 使用 /api/conversations/{id}/foreground/ws；终端实时 IO 使用 terminal stream。
删除语义DELETE /api/conversations/{id} 已由后端实现；会停止活跃 runtime、删除本地 conversation 目录并从列表流广播删除事件。
文档部署本页是纯静态 HTML，可直接用 GitHub Pages 从 docs/ 目录部署。

交互模型

Stellacode 2 是 Electron 客户端。Renderer 不直接持有 SSH tunnel 与文件保存能力，而是通过 preload 暴露的 window.stellacode2 调用 Electron main process；main process 再按 server settings 连接 Web Channel。

Renderer (React)
  ├─ JSON REST: window.stellacode2.request({ serverId, path, method, body })
  ├─ WebSocket URL: window.stellacode2.connectionInfo(serverId)
  ├─ Workspace upload/download: Electron main 处理 binary 与 save dialog
  └─ Settings / notification / updater: Electron IPC

Electron main
  ├─ Direct: baseUrl -> Stellaclaw Web Channel
  └─ SSH Proxy: ssh -L local_port:target_host:target_port -> Web Channel

Stellaclaw Web Channel
  ├─ REST: /api/models, /api/conversations/*, workspace, terminal lifecycle
  ├─ WS: /api/conversations/stream          (conversation list facts)
  ├─ WS: /api/conversations/{id}/foreground/ws
  └─ WS: /api/conversations/{id}/terminals/{terminal_id}/stream

关键边界：SSH Proxy 只是 Stellacode 连接 Stellaclaw 的方式；conversation 的工具执行位置由 status.remote / tool_remote_mode 决定，二者不要混用。

冗余与可能引发的歧义

点位现状风险建议
DELETE /api/conversations/{id} Web Channel router 已实现该路由；Stellacode 2 现在等待服务端删除成功后再更新本地列表。 删除会移除本地 conversation 目录、消息日志、session 状态、local workspace、已读状态和 cron 任务；fixed SSH 只卸载并移除本地 sshfs mountpoint，不删除远端目录。 保留为真实服务端删除语义；如后续需要“隐藏但不删除”，应新增单独 hide/archive 接口。
Conversation 状态事实源 /status 仍是按需快照接口；conversation summary 与 conversation stream 承载列表和 conversation 级状态；foreground stream 不再推 status / processing。 若新客户端重新从 foreground stream 推断列表状态，仍可能产生 running/idle 闪烁或旧状态回退。 列表与非消息级信息只信 /api/conversations/stream；当前聊天区只信 foreground messages / turn_progress / error。
消息分页接口 只保留 messages?offset&limit。WebSocket ack 和 message skeleton 都使用数组下标作为 message id。 客户端需要自己根据 current_message_id、已持有首尾 message id 和 offset/limit 计算补齐窗口。 不再提供 messages/after/{id} 或 messages/range，避免 gap repair、初始加载、历史加载出现多套分页语义。
Terminal REST 与 WS REST 只保留 terminal lifecycle；output/input/resize 统一走 terminal stream。 数据面只剩一套，offset、input 顺序和 resize 生效时机都由 WS 维护。 继续避免重新增加 REST output/input/resize fallback。
WebSocket subscription 风格 conversation list、foreground session 和 terminal 都使用路径表达订阅目标，连接后无需首帧订阅。 路径和 event type 已经表达来源；payload 不再携带冗余 subscription 字段。 新增 WS 时继续优先用路径语义，不要恢复通用 WS + 首帧订阅。
认证方式 HTTP 支持 Authorization: Bearer token 或直接等于 token；WS 支持 query ?token=。 实现宽松，文档若不明确会导致客户端互相复制非推荐用法。 推荐 HTTP 一律 Bearer；浏览器 WebSocket 使用 query token；直接 token header 只作为兼容。
Remote 语义 Electron server 有 connectionMode=direct/ssh_proxy；conversation status 有 remote。 把“客户端如何连服务端”误判为“工具在哪里执行”。 UI 文案中称前者为 Server Connection，后者为 Tool Execution Mode。

认证与传输

HTTP REST

推荐：

Authorization: Bearer <web_token>
Accept: application/json

JSON 请求体使用 Content-Type: application/json；workspace upload 使用 application/gzip。

WebSocket

浏览器无法稳定设置自定义 Authorization header，因此使用 query token：

ws://host:port/api/conversations/stream?token=<web_token>
ws://host:port/api/conversations/{id}/foreground/ws?token=<web_token>

所有非 /api/ 路径都会返回 404 not_found。预检 OPTIONS 返回 204。

REST API

Models

MethodPath用途响应
GET/api/models列出可选模型。{ models: WebModelSummary[] }

Conversations

MethodPath参数/Body用途
GET/api/conversationsQuery: offset=0, limit<=200分页列出 conversation summary。
POST/api/conversationsBody: { platform_chat_id?, model?, nickname? }创建或加载 conversation。
PATCH/api/conversations/{conversation_id}Body: { nickname? }更新 nickname。
POST/api/conversations/{conversation_id}/seenBody: { last_seen_message_id }标记已读；后端按最大 message id 单调合并。
DELETE/api/conversations/{conversation_id}—删除 conversation；停止活跃 runtime，清理本地 conversation 目录、已读状态、终端和 cron 任务，并向 list stream 广播 conversation_deleted。

Messages

MethodPath参数/Body用途
GET/api/conversations/{id}/messagesQuery: offset=0, limit<=200读取固定窗口消息页；客户端根据 message id 自行计算 offset。
GET/api/conversations/{id}/messages/{message_id}无读取消息详情、原始 message、渲染 items 与 attachments。
POST/api/conversations/{id}/messagesBody: { remote_message_id?, user_name?, text?, files? }投递用户消息或 slash command；返回 202 accepted。

Status & Workspace

MethodPath参数/Body用途
GET/api/conversations/{id}/status无读取详细状态快照：模型、remote、sandbox、background/subagent、usage。
GET/api/conversations/{id}/workspaceQuery: path?, limit<=1000列工作区目录。
GET/api/conversations/{id}/workspace/fileQuery: path, offset=0, limit_bytes读取文件内容或二进制 data URL。
POST/api/conversations/{id}/workspace/upload?path={dir}Body: gzip 压缩 tar archive上传文件/目录到 workspace。
GET/api/conversations/{id}/workspace/download?path={path}无下载 path 对应的 gzip tar archive。

Terminals

MethodPath参数/Body用途
GET/api/conversations/{id}/terminals无列出当前 conversation 的 terminal sessions。
POST/api/conversations/{id}/terminalsBody: { shell?, cwd?, cols?, rows? }创建 terminal，返回 TerminalSummary。
GET/api/conversations/{id}/terminals/{terminal_id}无读取 terminal summary。
DELETE/api/conversations/{id}/terminals/{terminal_id}无终止 terminal。

WebSocket API

Conversation list stream

KindPathHandshake用途
WS/api/conversations/stream?token=...连接后无需发送订阅消息左侧 conversation list 的唯一推荐实时事实源。
 
// initial
{ "type": "conversation_snapshot", "channel_id": "web", "conversations": [...] }

// later
{ "type": "conversation_upserted", "conversation_id": "web-main-000001", "conversation": {...} } // create/rename/model/remote/sandbox/session summary change
{ "type": "conversation_processing", "conversation_id": "...", "processing_state": "typing", "running": true }
{ "type": "conversation_turn_completed", "conversation_id": "...", "last_message_id": "42", "unread": true, "conversation": {...} }
{ "type": "conversation_seen", "conversation_id": "...", "seen": { "last_seen_message_id": "42", "updated_at": "..." } }
{ "type": "conversation_deleted", "conversation_id": "..." }

Foreground session stream

KindPathHandshake用途
WS/api/conversations/{id}/foreground/ws?token=...连接后无需发送订阅消息当前打开 conversation 的消息、turn progress 和错误。
 
// server -> client
{ "type": "subscription_ack", "reason": "subscribed", "next_message_id": "12", "current_message_id": "11", "turn_progress": null }
{ "type": "messages", "conversation_id": "...", "messages": [...] }
{ "type": "turn_progress", "turn_id": "...", "phase": "thinking|working|done|failed", "activity": "...", "plan": {...}, "progress": {...} }
{ "type": "error", "severity": "error", "message": "...", "can_continue": true }

Terminal stream

KindPath帧类型用途
WS/api/conversations/{id}/terminals/{terminal_id}/stream?offset=...Binary + JSON TextPTY 实时输入输出，支持 offset replay。

{ "type": "attached", "terminal_id": "...", "offset": 0, "replay_start_offset": 0, "next_offset": 1024, "running": true }
{ "type": "dropped", "dropped_bytes": 4096, "buffer_start_offset": 4096 }
{ "type": "exit", "terminal_id": "..." }
{ "type": "detached", "terminal_id": "...", "reason": "slow_client" }

// server -> client Binary
<raw PTY output bytes>

// client -> server Binary
<raw stdin bytes>

// client -> server Text controls
{ "type": "resize", "cols": 120, "rows": 30 }
{ "type": "attach", "offset": 8192 }
{ "type": "ping" }

Stellacode 2 当前使用方式

  • apps/stellacode2/src/lib/api.js 是 renderer 侧 Web Channel API wrapper。
  • apps/stellacode2/electron/main.cjs 负责 server settings、Direct/SSH Proxy、请求超时、workspace 二进制上传下载、系统通知、更新。
  • App.jsx 维护两个长连接：conversation list stream 和当前 foreground session stream；前者承载 conversation 级事实，后者只承载当前消息、turn progress 和错误。
  • workspace upload/download 不走通用 api() wrapper，因为需要 gzip binary body、tar 解析和 native save dialog。
  • terminal dock 使用 terminal WebSocket 承载 PTY output/input/resize；REST 只保留 terminal 生命周期管理。
如果后续引入第三方 Web client，建议先实现同样的三条事实源分工：list stream 管侧栏，foreground stream 管当前聊天，terminal stream 管 PTY；不要再增加轮询状态推断。

GitHub Pages 部署方式

本页是纯静态文件，已经放在 docs/web-channel-api.html。部署方式：

  1. Push 到 GitHub。
  2. 进入仓库 Settings → Pages。
  3. Source 选择 Deploy from a branch。
  4. Branch 选择 main，Folder 选择 /docs。
  5. 保存后访问：https://<owner>.github.io/<repo>/web-channel-api.html。

如果你想把它作为 docs 首页，访问 / 时会进入 docs/index.html，该文件会链接到本页。
